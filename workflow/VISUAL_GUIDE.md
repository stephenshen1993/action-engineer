# Action Engineer 视觉指南

## 核心原则

视觉服务于内容。

不追求炫酷。

不追求AI感。

不追求科技感。

优先保证：

- 真实感
- 一致性
- 长期可持续

标题、封面、头图不要重复表达同一个信息。

三者分工：

- 标题负责制造停顿
- 封面负责提供点击理由
- 头图负责提供现场感和情绪

---

# 封面

职责：

让用户停下来。

封面负责点击，不负责完整表达现场。

封面可以放大矛盾、强化冲突，但不要承担正文解释任务。

## 默认模板（80%文章）

元素：

- 真实工位
- 暖色灯光
- 夜晚环境
- 电脑/笔记本
- 生活痕迹

要求：

- 文字 2~4 行
- 大字
- 少元素
- 可以直接使用反直觉标题
- 优先具体冲突，不要过度文学化或抽象化
- 不出现机器人
- 不出现未来城市
- 不出现赛博朋克
- 避免科技海报感

封面文案优先使用具体动作和具体冲突。

优先：

- `明天上线，我却重做了`
- `做了两周，我又推翻了`
- `代码快写完了，我却慌了`

慎用：

- `差一点`
- `做不下去了`
- `说服不了`

抽象表达可以有气质，但通常不如具体冲突更容易让用户停下来。

---

# 封面（重大节点）

适用：

- 世界观升级
- 人生转折
- 重大认知变化

允许：

- 人物
- 剪影
- 电影感

禁止：

- AI导师感
- 成功学海报感

---

# 头图

职责：

进入文章后的情绪铺垫。

头图负责现场感，不负责点击。

原则：

- 不重复标题
- 不解释内容
- 只提供情绪
- 服务文章核心现场，而不是服务主题关键词

优先场景：

- 工位
- 窗边
- 夜景
- 路灯
- 咖啡馆
- 地铁

允许无文字。

选择头图时，先判断文章真正发生在哪里。

如果核心现场是朋友聊天，就优先表现“一个人在认真回复消息”，而不是强行表现 AI、代码、程序员工位或科技感元素。

头图不是关键词配图。

它应该让读者进入文章发生的那个时刻。

当文章讨论 AI Agent 协作时，可以优先表现：

- 程序员处于观察、后仰、Review 或等待状态
- AI 或屏幕任务在持续执行
- 人物不再每一步都亲自操作
- 人在看活，而不是亲自干活

重点表达工作关系的变化，而不是展示工具界面。

---

# 插图

默认不使用。

仅在确有必要时使用。

每篇最多2张。

职责：

补充情绪或场景。

不是解释内容。

---

# 配乐

选音乐前先判断文章结尾属于哪一类。

## A 类：问题获得解决

适用：

- 想明白了
- 获得答案
- 完成成长
- 问题已经被解决

可以考虑：

- `Nuvole Bianche`
- `Experience`
- `Una Mattina`

## B 类：问题仍然悬着

适用：

- 刚发现问题
- 开始怀疑
- 开始慌张
- 问题尚未解决
- 结尾停在不确定里

这类文章应选择更克制、更悬而未决、更偏思考感的音乐。

避免使用明显带有“解决问题”“获得答案”气质的配乐。
