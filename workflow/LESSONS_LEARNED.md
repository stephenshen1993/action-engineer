# 创作复盘

## 2026-06-02 第17篇《本来想解决一个问题，结果事情越来越不对劲》

### 文章质量

文章质量提升的关键通常不是增加观点，而是增加：

- 故事
- 情绪
- 真实过程

减少：

- 总结
- 方法论
- 说教

优先记录：

- 事情开始不对劲的时刻
- 困惑出现的时刻
- 怀疑出现的时刻
- 荒诞感出现的时刻

这些通常比最终结论更有价值。

工程师共鸣来源：

> 卧槽，这不就是我吗

而不是：

> 这个人懂得真多

当文章开始变成长篇认知分析时，优先追问：

> 当时发生了什么

而不是：

> 最后得出了什么结论

### 标题选择

标题优先级：

> 故事感 > 悬念感 > 观点表达

不要按散文标准选标题。优先按抖音点击逻辑选标题。

### 停止重写

出现以下情况立即进入发布流程：

- 核心矛盾明确
- 情绪真实
- 开始纠结字句
- 开始讨论 V3、V4、V5

### 长期协作风险

ChatGPT 容易出现“基线版本锁死”：

- 死抱旧标题
- 死抱旧结构
- 死抱旧结论

后续协作中发现此问题时，直接放弃旧基线，以最新结论为准。

## 2026-06-04 第18篇《我明明想找个助手，最后却当了两天监工》

### 选题获取

本次没有从抽象观点出发。

选题直接来自最近 48 小时真实经历：

- Codex 重装
- 自动审查模式
- Claude Code 落地
- 网络质量评估需求 Review

验证：

> 真实现场比刻意寻找选题更容易形成内容。

### 工具题材风险

创作过程中多次差点滑向：

> Claude Code 好用，所以写工具测评。

最终确认：

工具只是故事。

变化才是主题。

当选题涉及 AI 工具时，不要写工具本身，优先写工具带来的变化。

### 内容阶段变化

第 18 篇标志着项目进入新的内容阶段。

前期内容：

> AI 改变我的思维。

当前内容：

> AI 开始参与我的工作。

后续可扩展方向：

- AI 同事感
- AI 信任建立
- AI 协作模式
- Agent 时代的工程师
- 控制欲与放权
- AI Review 体验
- 工程师如何与 Agent 协作

### 视觉经验

再次确认：

- 封面负责点击
- 头图负责现场感

不要混用。

本次最终头图方向：

> 程序员后仰坐在工位前，AI 持续执行任务，人物处于观察状态。

表达：

> 终于不用每一步都自己干了。

### 原生表达

本次最有价值的真实表达：

> 原来不是它不会干。
>
> 是我一直没让它干。

真实吐槽往往比总结性观点更有力量。

创作时优先保留真实表达。

## 2026-06-04 第19篇《有时候，我真希望它干慢一点》

### 选题来源

本次选题来自最近真实经历：

- 折腾 Codex 自动审查模式
- 深度使用 Agent 协作
- 多个 Agent 并行工作
- 不断授权、Review、拍板

最初讨论过的方向包括：

- AI 时代重新变成新人
- 学习 Agent 的折腾感
- AI 带来的新工作方式

最终发现这些都不是核心矛盾。

### 核心矛盾

错误方向：

- AI 很强
- Agent 很快
- 我跟不上 AI
- 我看不懂 AI

正确方向：

> Agent 提升了执行速度，但责任没有转移。

更准确地说：

> 执行的人变了。
>
> 负责的人没变。

瓶颈开始从执行能力，转向判断能力、Review 能力和拍板责任。

### 重要纠偏

曾经尝试把文章写成：

> 我开始对没看懂的东西负责。

后来确认这不准确。

问题不是看不懂。方案、设计和关键逻辑依然会 Review，必要时会让 Agent 解释到自己听懂为止。

真正的问题是：

> 很多事情已经不是自己亲手完成的了。

因此核心矛盾应调整为：

> 执行权转移，责任权未转移。

### 关键证据链

本篇最有活人感的观察：

最近几天与 ChatGPT 共创时，说得最多的话是：

- 没问题
- 继续推进
- 下一步

这与使用 Agent 时的行为几乎一致：

```text
Agent 提方案
→ Review
→ 继续推进

Agent 干完活
→ Review
→ 继续推进

Agent 申请执行
→ Review
→ 继续推进
```

这是整篇文章最关键的证据链。

### 创作经验

不要从观点开始，要从现场开始。

最终采用的开头不是：

> 我发现自己变成了瓶颈。

而是：

> 昨天晚上，我第一次希望 Agent 干慢一点。

真实发生过的别扭瞬间，通常比结论更有价值。

本篇的别扭感来自：

- 刚进入心流，Agent 来申请授权
- 刚切回工作，Agent 又来申请授权
- 不断被打断
- 最后意识到，限制 Agent 效率的人变成了自己

### 表达控制

不要过早升华。

创作过程中出现过这些总结型表达：

- 判断能力变稀缺
- 价值迁移
- Agent 时代的新能力

最终全部收掉。

保留观察，让读者自行得出结论。

结尾优先“发现”，不要“总结”。

错误收尾：

> 判断能力比执行能力更重要。

正确方向：

> 最近我说得最多的话好像都是：
>
> 没问题。
>
> 继续推进。
>
> 下一步。

然后停住。

### 标题选择

讨论过的标题：

- 现在最大的瓶颈，居然是我自己
- Agent 干得太快，我有点跟不上了
- 我发现自己越来越像审批人了
- 最近我一直在点同意

最终选择：

> 有时候，我真希望它干慢一点

原因：

- 反直觉
- 有悬念
- 不像 AI 教程
- 不像成功学
- 与正文矛盾完全一致

### 方法论更新

当核心矛盾已经明确后，禁止继续围绕同一问题无限深挖。

本次再次验证：

> 80 分已发布文章，远好于 95 分未发布文章。

创作后期应尽快从选题讨论，切换到：

> 正文 → 头图 → 封面 → 发布。

## 2026-06-05 阶段性数据复盘

复盘范围：

> 2026-05-16 ～ 2026-06-05

本次结合了已发布文章的数据表现，包括播放量、点赞、评论、收藏、详情页进入率和完播率。

### 表现更好的内容形态

表现最好的内容并不是 AI 功能介绍。

更有效的是：

> 真实工作现场 + 明确冲突 + 真实情绪。

代表文章包括：

- `那我这一下午到底在查什么？`
- `我明明想找个助手，最后却当了两天监工`
- `有时候，我真希望它干慢一点`
- `老板一句话，我研究了快两周`

用户未必关心 AI 本身。

用户更关心：

> 一个真实工程师在现实工作中的困惑、挣扎、成长与变化。

### 选题入口

不要从 AI 开始。

优先从真实现场开始。

不要从观点开始。

优先从具体事件开始。

不要记录工具。

优先记录人的变化。

如果一篇文章只能回答“这个工具有什么功能”，通常不够。

如果一篇文章能回答“这个工具让我的工作方式、判断方式、情绪状态发生了什么变化”，就更接近项目主线。

### 当前最大风险

项目过去最大的风险是：

> 无法开始。

当前最大的风险开始变成：

> 无法结束。

具体表现：

- 无限优化标题
- 无限优化开头
- 无限优化表达
- 无限优化头图
- 无限优化封面

这些优化看起来都是为了质量，但会让单篇耗时持续增长，最后伤害长期持续性。

### 发布标准

文章满足以下条件即可进入发布流程：

- 有真实现场
- 有明确矛盾
- 有情绪变化
- 可顺畅读完

达到标准后应优先发布，避免继续无限优化。

这不是降低标准，而是把标准从“局部最好”，切回“长期可持续”。

### 已验证方向

较容易引发共鸣的主题：

- 长期准备，却迟迟没有行动
- 太想找到正确答案，反而开始不了
- 工程师容易过度系统化
- AI 带来的职业焦虑与行动变化
- AI 如何改变工程师解决问题的方式
- AI Agent 如何参与真实工作
- 工程师如何从亲自控制每一步，转向放权、判断和 Review
- Agent 执行变快后，人如何承接 Review 压力和责任边界
- 投资和长期价值决策中的自我合理化
- 普通工程师能力边界的变化

### 阶段风险

当前最大风险已经从“无法开始”，变成“无法结束”。

需要持续警惕：

- 无限优化标题
- 无限优化开头
- 无限优化表达
- 无限优化头图
- 无限优化封面
- 用局部质量提升拖高单篇耗时

仍然要警惕再次陷入：

> 先把系统设计正确，再开始行动。

## 2026-06-06 第20篇《买了再研究》

### 选题发现

本篇再次验证：

> 好选题经常不是策划出来的，而是从真实反应里冒出来的。

开始讨论的方向可能很大，比如投资策略、卖出规则、产品化、资产化。

但真正有内容价值的入口，往往是聊天中突然出现的真实惊讶、困惑、别扭感或认知冲突。

后续创作时要优先捕捉这些信号，而不是急着寻找大道理。

### 找到矛盾后及时写

当核心矛盾已经出现时，继续深挖容易把文章带抽象。

常见风险：

```text
真实现场
↓
行为分析
↓
心理学
↓
方法论
↓
说教
```

更适合本项目的路径是：

```text
真实现场
↓
真实困惑
↓
真实变化
```

找到矛盾后优先写，不要继续哲学化。

### 变化结构

本篇有效结构不是“观点 + 例子 + 总结”，而是：

```text
不理解
↓
能理解
↓
自己也经历过
↓
后来变了
```

后续写真实经历时，可以优先寻找：

> 我以前怎么想。
>
> 现在怎么想。

变化过程比结论更能承载真实感。

### 标题经验

标题不负责总结内容。

标题职责是：

- 制造停顿
- 制造好奇
- 制造反差

短标题通常比完整标题更适合抖音长文。

优先选择：

> 具体动作 + 反直觉

避免把标题写成抽象总结。

### 视觉职责

本篇再次确认：

- 标题负责制造停顿
- 封面负责提供点击理由
- 头图负责提供现场感和情绪

三者不要重复表达同一个信息。

头图不解释文章，不复述标题。

封面可以放大矛盾，但不要做成说教海报。

### 协作分工

当前较稳定的协作方式：

- ChatGPT 负责挖素材、找选题、找矛盾、搭结构
- Claude 负责后期去 AI 味、修断句、修连接词、优化节奏

Claude 不适合在后期重写观点。

当文章核心已经明确后，Claude 更适合做语言打磨，而不是重新定义文章。

### 账号成熟度

前期容易优先寻找 AI 话题。

当前更应该优先寻找真实经历中的冲突。

素材可以来自投资、工作、家庭、内容创作或 AI 协作。

只要最终落点仍然是：

> 普通工程师在变化。

就符合项目定位。

不需要强行绑定 AI。

## 2026-06-07 第21篇《今天下午，我差点没忍住》

### 镜像型选题

本篇验证了一种新的高质量选题来源：

> 在别人身上看见过去的自己。

这类选题不一定来自“今天发生了什么”或“今天学到了什么”，而是来自某个瞬间：

- 看见曾经的犹豫
- 看见曾经的观望
- 看见曾经踩过的坑
- 看见曾经的认知局限
- 突然很想把别人拉出某个状态

创作重点不是教育别人，而是描述：

> 为什么这一幕会触动自己。

这类内容天然适合 Action Engineer，因为它记录的不是工具知识，而是一个工程师在变化之后，重新看见过去的自己。

### 从工具分享转向情绪与成长

本篇前期容易滑向：

- AI 工具分享
- 模型比较
- 订阅成本讨论
- 使用经验推荐

这些信息可以作为事件载体，但不应该成为文章核心。

真正有价值的部分是：

> 为什么我突然这么想把别人拉起来。

当一篇 AI 相关选题开始变成工具推荐时，要及时回到人的变化：

- 我为什么在意
- 我为什么着急
- 我看见了过去哪个阶段的自己
- 这件事暴露了什么新的关系或心态

### 具体情绪强于抽象观点

本篇最有力量的表达不是“AI 很重要”或“效率提升很大”，而是具体的原话、原场景和原反应。

未来写作优先寻找：

- 原话
- 原场景
- 原反应

不要急着把它们翻译成总结性观点。

真实表达通常比抽象判断更有传播力。

### 标题、摘要、正文分工

本篇再次确认，抖音长文不一定要让标题独自承担全部悬念。

更稳定的分工是：

- 标题负责故事
- 摘要负责钩子
- 正文负责情绪

标题优先像一个真实经历，而不是一个纯悬念句。

摘要可以补充悬念，但不要让标题失去活人感。

### 视觉经验

头图应服务文章核心现场，而不是服务主题关键词。

如果文章真正发生在一次朋友聊天里，最佳头图就不一定是 AI、代码、工位或科技元素。

更合适的方向可能是：

> 一个人认真回复消息。

判断头图时，先问：

> 这篇文章真正发生在哪个时刻？

然后再决定画面。

### 长期提醒

不要只盯着：

> 今天发生了什么。

也要留意：

> 今天谁让我想起了曾经的自己。

这可能是未来高质量内容的重要来源。
