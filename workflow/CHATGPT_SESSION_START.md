# ChatGPT 网页版新会话启动流程

> 职责：记录每次在 ChatGPT 网页版开启创作新会话时，应该上传哪些文件、怎么发起对话、如何让 ChatGPT 先追问素材。
>
> 不记录：完整写作方法、单篇复盘、发布后归档流程。

## 使用场景

绝大多数时候，作者并不是已经想清楚一篇文章要写什么。

更常见的情况是：

- 只有一个模糊现场
- 只有一段真实情绪
- 只是觉得某件事有点不对劲
- 还不知道核心矛盾在哪里
- 需要 ChatGPT 通过追问把选题挖出来

因此，新会话默认不要让 ChatGPT 立刻写正文。

默认任务是：

> 先追问，后写稿。

## 默认上传文件

每次新开 ChatGPT 网页版创作会话，默认提供 3 个文件：

1. `PROJECT_CONTEXT.md`
2. `CURRENT_STATUS.md`
3. `workflow/CREATION_GUIDE.md`

这 3 个文件分别解决：

- 项目长期定位是什么
- 当前阶段最重要的提醒是什么
- 日常创作和 Review 应该遵守什么规则

不要默认上传所有历史复盘和全部已发布文章。

## 启动词

上传默认文件后，发送以下启动词：

```text
你现在参与 Action Engineer 项目的日常创作。

我已经提供了：
1. 项目长期定位：PROJECT_CONTEXT.md
2. 当前阶段状态：CURRENT_STATUS.md
3. 创作规则：CREATION_GUIDE.md

这次你的职责是：
- 先通过追问帮我从真实经历里挖素材
- 找到最值得写的核心矛盾
- 不要急着总结观点
- 不要立刻写正文
- 不要写成 AI 教程、工具测评、成功学或方法论文章
- 优先保留真实现场、真实情绪和人的变化
- 当素材和矛盾足够清楚后，再帮我形成一版可发布初稿
- 达到 80 分可发布状态后，提醒我停止优化

多数情况下，我一开始并没想清楚要写什么。
你先用聊天方式追问我，目标不是马上写稿，而是帮我找到：
1. 真实现场
2. 当时最强的情绪
3. 最值得写的矛盾
4. 这件事背后我发生了什么变化

每次最多问 3 个问题。
```

## 素材输入模板

如果已经有一点素材，可以继续补充：

```text
真实现场：
当时最真实的感受：
核心矛盾：
最想表达的一句话：
必须保留的细节：
需要规避的隐私风险：
```

如果暂时没有完整素材，只要先描述一个模糊现场即可。

例如：

```text
我现在只有一个模糊感觉：

今天发生了一件事，大概是……
我当时有点……
但我还说不清楚这件事到底值不值得写。

你先追问我，不要立刻写正文。
```

## 追问阶段目标

ChatGPT 追问时，优先帮作者确认：

1. 当时到底发生了什么。
2. 哪个瞬间让人停了一下。
3. 最真实的情绪是什么。
4. 这个情绪背后有没有矛盾。
5. 这件事是否暴露了作者的变化。
6. 是否适合写成一篇真实经历型文章。

追问阶段不要急着：

- 起标题
- 搭完整结构
- 输出金句
- 总结方法论
- 做价值升华
- 直接生成正文

## 进入写稿的条件

满足以下条件后，再让 ChatGPT 写初稿：

- 有一个具体现场
- 有一个真实情绪
- 有一个明确矛盾
- 能说清楚这件事和作者变化有什么关系
- 已经知道哪些隐私、客户或同事信息需要规避

可以发送：

```text
现在可以进入初稿。

请按 Action Engineer 的创作规则写一版可发布初稿：
- 保留真实现场
- 保留真实情绪
- 不要过早升华
- 不要写成教程或方法论
- 读起来像一个人真实经历过
- 结尾停在发现或变化上，不要强行总结
```

## 按需补充文件

只有在需要时，再补充以下文件：

| 场景 | 补充文件 |
| --- | --- |
| 避免和历史内容重复 | `content/published/INDEX.md` |
| 参考某篇文章气质 | 对应的已发布文章全文 |
| 制作封面、头图或配乐 | `workflow/VISUAL_GUIDE.md` |
| 查找历史经验 | `workflow/LESSONS_LEARNED.md` 的相关片段 |

`workflow/LESSONS_LEARNED.md` 默认不要全文上传。

它是回查资料，不是每次新会话的完整上下文。

## 结束前交接

当文章基本成型后，让 ChatGPT 输出一份交接信息，方便后续给 Claude 打磨或交给 Codex 归档：

```text
请输出本篇交接信息：

标题：
正文状态：
核心矛盾：
真实现场：
最重要的情绪：
必须保留的原话或细节：
隐私规避提醒：
封面方向：
头图方向：
建议标签：
是否达到 80 分可发布：
还需要修改的一处阻塞项：
```

如果没有阻塞项，要求 ChatGPT 明确写：

```text
阻塞项：无，可以发布。
```

## 最重要提醒

新会话的第一目标不是写出漂亮文章。

第一目标是把真实素材问出来。

继续记住：

> 先追问，后写稿。
>
> 先真实，后深刻。
>
> 80 分已发布文章，远好于 95 分未发布文章。
