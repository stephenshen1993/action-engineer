# 发布前单次 Review

## 使用规则

这是文章发布前唯一一次正文 Review。

只判断是否存在阻塞发布的问题。不要继续寻找可选优化项，不要提供多个改写版本。

## 输入

```text
核心矛盾：
正文：
```

## 只检查四类阻塞项

1. 是否存在事实错误？
2. 是否存在隐私、客户或同事关系风险？
3. 是否存在普通读者无法理解的明显歧义？
4. 正文是否偏离本篇核心矛盾？

## 输出格式

```text
结论：发布 / 修改一次后发布

阻塞项：
- 没有则写“无”
- 有则逐条指出，并给出最小修改建议

每周复盘观察：
- 可选
- 最多记录 3 条
- 不触发当天继续修改
```

## 不再阻塞发布

以下问题不触发当天继续修改：

- 还能更顺
- 还能更有情绪
- 可能略有 AI 味
- 或许可以换一种结构
- 标题也许存在更优解
