# Action Engineer 轻量创作工作流设计

## 背景

当前每天完成一篇抖音长文通常耗时超过 3 小时。

结合最近文章的创作过程，主要瓶颈不是缺少选题，也不是无法形成初稿，而是初稿达到可发布状态后仍持续修改：

- 反复调整标题
- 反复确认是否有 AI 味
- 继续寻找更顺的表达
- 预判读者是否会误解
- 在没有新增事实的情况下持续 Review

项目仓库已经沉淀了定位、动态状态、写作规则和文章索引，但存在入口分散、重复文件和失效引用。整理时需要控制范围，避免工作流优化本身再次挤占创作时间。

## 目标

用最小改动支持一个 7 天试运行：

1. 将每天创作耗时压到 90 分钟以内。
2. 初稿后最多执行一次集中 Review 和一次集中修改。
3. 将 Review 从开放式优化改为阻塞项检查。
4. 保留 ChatGPT 负责深聊与成稿、Codex 负责归档与长期资产维护的分工。
5. 统一 ChatGPT 获取项目上下文的入口，减少手动补充背景。

## 非目标

本轮不引入：

- Agent 编排
- 自动化同步
- API 集成
- 数据分析系统
- 多平台内容矩阵
- 复杂文章版本管理

## 文件调整

### 保留并明确职责

| 文件 | 职责 |
| --- | --- |
| `PROJECT_CONTEXT.md` | 稳定项目背景、账号定位和长期方向 |
| `CURRENT_STATUS.md` | 唯一动态状态文件 |
| `workflow/writing_guide.md` | 写作规则统一入口 |
| `CHATGPT_PROJECT_BRIEF.md` | 上传到 ChatGPT Project 的精简上下文包 |
| `workflow/templates/article_creation_card.md` | 每篇文章开始前填写的短卡片 |
| `workflow/article_review_prompt.md` | 只检查阻塞发布问题的单次 Review Prompt |
| `workflow/action_engineer_current_workflow_v_1.md` | 更新后的日常创作与归档流程 |

### 清理重复与失效入口

- 将 `CURRENT_STATE.md` 标记为废弃，动态信息只维护在 `CURRENT_STATUS.md`。
- 修复 `CHATGPT_CONTEXT.md` 对不存在的 `workflow/writing_guide.md` 的引用。
- 用 `workflow/writing_guide.md` 统一指向现有的风格、正面样例、负面规则和视觉规范。
- 删除工作流中不存在或已不再使用的 `drafts`、`article_index` 等旧描述。

## 每日流程

```text
5 分钟：填写单篇创作卡
30 分钟：ChatGPT 深聊并形成可发布初稿
15 分钟：执行一次阻塞项 Review
20 分钟：集中修改一次
5 分钟：从最多 3 个标题中选择 1 个
10 分钟：交给 Codex 归档
5 分钟：预留缓冲
```

总耗时上限为 90 分钟。超过上限时，优先发布已经满足阻塞项检查的版本。

## 单篇创作卡

开始写作时只记录：

```text
真实现场：
核心矛盾：
最想表达的一句话：
必须保留的细节：
需要规避的隐私风险：
```

它用于缩短启动时间，不承担长期知识库职责。

## Review 停止规则

Review 只允许拦截四类问题：

1. 事实错误。
2. 隐私、客户或同事关系风险。
3. 普通读者无法理解的明显歧义。
4. 正文偏离本篇核心矛盾。

以下问题可以记录到每周复盘，但不得阻塞当天发布：

- 还能更顺
- 还能更有情绪
- 可能略有 AI 味
- 或许可以换一种结构
- 标题也许存在更优解

标题候选最多 3 个，只做一次选择。

## 上下文同步

ChatGPT Project 中上传 `CHATGPT_PROJECT_BRIEF.md`。

精简上下文包只包含：

- 项目一句话定位
- 当前阶段
- 近期内容方向
- 写作底线
- 已发布内容索引入口
- 本周提醒

由 Codex 在每周复盘或内容方向明显变化时更新。日常创作不维护上下文包。

## 试运行验证

连续试运行 7 天，只记录：

| 指标 | 目标 |
| --- | --- |
| 每天总耗时 | 不超过 90 分钟 |
| 初稿后修改次数 | 不超过 1 次 |
| 是否完成发布 | 是 / 否 |

试运行期间不因为单篇数据表现调整工作流。

## 成功标准

7 天后满足以下条件，即保留该流程：

1. 大部分文章能在 90 分钟内完成。
2. 初稿后的反复打磨明显减少。
3. 稳定输出没有中断。
4. 上下文同步不再成为每天的额外负担。
