# Content 目录说明

这里存放内容资产。

## 目录职责

| 目录 | 职责 | 维护规则 |
| --- | --- | --- |
| `published/` | 已发布文章正文和内容地图 | 每篇文章一个 Markdown 文件，并更新 `INDEX.md` |
| `assets/published/` | 已发布文章实际使用的图片 | 只保留最终封面、头图和正文实际使用插图 |
| `ideas/` | 尚未进入创作阶段的真实现场 | 简短记录，不追求完整 |

## 归档原则

1. 已发布文章才进入 `published/`。
2. 过程稿、过程封面和过程头图默认不归档。
3. 素材目录名与文章文件名保持一致。
4. `published/INDEX.md` 只做内容地图，不保存文章全文。
5. 单篇创作复盘写入 `workflow/LESSONS_LEARNED.md`，不写入内容目录。

当前阶段重点是方便回看和持续更新，不需要复杂内容系统。
